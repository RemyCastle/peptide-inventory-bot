WHAT YOU NEED TO KNOW
  Multi-shop Telegram bot: each Telegram group (or personal chat) = one shop.
  Customers browse catalog, cart, checkout, pay; admin confirms payment then
  stock drops. Owners use OWNER_IDS in .env for global access.

  Related but SEPARATE: peptide_telegram_bot = dosing calculator only.

HOW TO RUN
  cd C:\Users\Remy\peptide_inventory_bot
  copy .env.example .env   (first time)
  # set TELEGRAM_BOT_TOKEN and OWNER_IDS
  start.bat
  # or: python bot.py
  Only ONE runner at a time (local OR cloud) or Telegram conflicts.

LOGINS / IDs TO TRACK
  Bot token:          .env → TELEGRAM_BOT_TOKEN
  Owner Telegram IDs: .env → OWNER_IDS (comma-separated numeric IDs)
  Shop admins:        added in-bot by owner/admin (admins table)
  Brand:              BRAND_NAME in .env (e.g. UnicornFartzzBot)
  No password logins — all identity is Telegram.

HOW COLLAB WORKS (customer story)
  1. Shop A admin: Admin → Collaborations → Create invite link
  2. Send link to Shop B admin → they open & accept
  3. Shop A picks products from B + markup (e.g. +15%)
  4. Customer in Shop A sees A products + B products at marked-up prices
  5. Customer pays Shop A payment methods for entire order
  6. On confirm paid: stock drops in A and B for their lines
  7. Shop A admin settles payment to Shop B for B’s portion
